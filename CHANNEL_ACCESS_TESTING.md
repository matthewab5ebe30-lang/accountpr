# 🔐 КАК ПРОВЕРИТЬ ДОСТУП В КАНАЛ

## ⚠️ ВАЖНО: Почему администратор не будет удален из канала

Если вы **Создатель** или **Администратор** канала, то метод `ban_chat_member` **НЕ СРАБОТАЕТ** на вас!

Telegram API не даёт боту удалять создателей и админов канала. Это защита от случайного блокирования.

## ✅ ПРАВИЛЬНЫЙ СПОСОБ ТЕСТИРОВАНИЯ:

### Вариант 1: Создать второй тестовый аккаунт (ЛУЧШИЙ)

1. **Создайте второй Telegram аккаунт** (на другом телефоне/номере)
2. **Отправьте /start боту с тестового аккаунта**
3. **НЕ добавляйте тестовый аккаунт в админы канала!**
4. **Создайте тестовую подписку для этого аккаунта:**

```bash
python test_subscription_cycle.py --create-test
# Введите Telegram ID тестового аккаунта
# Введите срок: 2 минуты
```

5. **Подождите 2+ минуты**
6. **Запустите проверку:**
```bash
python test_subscription_cycle.py --check-expired
# ИЛИ в боте: /test_expired (от админского аккаунта)
```

7. **Проверьте что тестовый аккаунт:**
   - ❌ НЕ МОЖЕТ зайти в канал
   - ✅ Получил уведомление "Подписка истекла"
   - ✅ Может снова купить подписку

```bash
Друг → Нажимает "Купить подписку" → Оплачивает

python3 << 'EOF'
import asyncio
import asyncpg
from datetime import datetime, timedelta

async def activate():
    conn = await asyncpg.connect('postgresql://postgres:postgres@localhost:5432/subscription_bot')
    
    telegram_id = 123456789  # ID друга
    
    # Получаем user_id
    user = await conn.fetchrow("SELECT id FROM users WHERE telegram_id = $1", telegram_id)
    
    # Создаем платёж
    payment = await conn.fetchrow("""
        INSERT INTO payments (user_id, payment_id_yookassa, amount, currency, status)
        VALUES ($1, 'test_manual_' || $1, 100.00, 'RUB', 'succeeded')
        RETURNING id
    """, user['id'])
    
    # Активируем подписку на 2 минуты
    end_date = datetime.now() + timedelta(minutes=2)
    await conn.execute("""
        INSERT INTO subscriptions (user_id, payment_id, status, start_date, end_date)
        VALUES ($1, $2, 'active', NOW(), $3)
    """, user['id'], payment['id'], end_date)
    
    print(f"✅ Подписка активирована до {end_date}")
    await conn.close()

asyncio.run(activate())
EOF
```

## ⚡ БЫСТРАЯ КОМАНДА ДЛЯ АКТИВАЦИИ ТЕСТОВОЙ ПОДПИСКИ:

```bash
cd /workspaces/accountpr
source .venv/bin/activate

# Замените 123456789 на реальный Telegram ID
python3 << 'EOF'
import asyncio
import asyncpg
from datetime import datetime, timedelta

async def quick_test_sub():
    telegram_id = int(input("Telegram ID: "))
    minutes = int(input("Срок в минутах: "))
    
    conn = await asyncpg.connect('postgresql://postgres:postgres@localhost:5432/subscription_bot')
    
    # Получаем/создаем пользователя
    user = await conn.fetchrow("SELECT id FROM users WHERE telegram_id = $1", telegram_id)
    
    if not user:
        print("❌ Пользователь не найден! Попросите отправить /start боту")
        await conn.close()
        return
    
    # Создаем платёж
    payment = await conn.fetchrow("""
        INSERT INTO payments (user_id, payment_id_yookassa, amount, currency, status)
        VALUES ($1, 'test_' || NOW()::text, 100.00, 'RUB', 'succeeded')
        RETURNING id
    """, user['id'])
    
    # Активируем подписку
    end_date = datetime.now() + timedelta(minutes=minutes)
    await conn.execute("""
        INSERT INTO subscriptions (user_id, payment_id, status, start_date, end_date)
        VALUES ($1, $2, 'active', NOW(), $3)
    """, user['id'], payment['id'], end_date)
    
    print(f"\n✅ Тестовая подписка создана!")
    print(f"⏰ Истекает: {end_date.strftime('%Y-%m-%d %H:%M:%S')}")
    print(f"⏳ Через {minutes} минут\n")
    
    await conn.close()

asyncio.run(quick_test_sub())
EOF
```

## 🆘 Если что-то не работает:

### Проблема: Бот не может удалить пользователя
**Причина:** Бот не админ канала или у него нет прав
**Решение:** 
1. Добавьте бота (@MargoClubBot) в админы канала
2. Дайте права: "Ban users"

### Проблема: Даже после истечения пользователь видит канал
**Причина:** Возможно это админ
**Решение:** Проверьте роль через настройки канала

### Проблема: Не работает invite link
**Причина:** У бота нет прав "Invite users via link"
**Решение:** Дайте боту это право в настройках админа
