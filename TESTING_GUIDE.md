## 📋 СПОСОБ 3: Изменить дату подписки в БД напрямую

```bash
cd /workspaces/accountpr
source .venv/bin/activate

python3 << 'EOF'
import asyncio
import asyncpg
from datetime import datetime, timedelta

async def set_subscription_expiry():
    conn = await asyncpg.connect('postgresql://postgres:postgres@localhost:5432/subscription_bot')
    
    telegram_id = int(input("Telegram ID: "))
    minutes = int(input("Истекает через минут: "))
    
    end_date = datetime.now() + timedelta(minutes=minutes)
    
    result = await conn.execute("""
        UPDATE subscriptions s
        SET end_date = $1
        FROM users u
        WHERE s.user_id = u.id
          AND u.telegram_id = $2
          AND s.status = 'active'
    """, end_date, telegram_id)
    
    print(f"✅ Обновлено: {result}")
    print(f"⏰ Подписка истечет: {end_date.strftime('%Y-%m-%d %H:%M:%S')}")
    
    await conn.close()

asyncio.run(set_subscription_expiry())
EOF
```

## 📊 Проверка уведомлений о скором истечении

### Вариант 1: Изменить REMINDER_DAYS в config.py

Установите `REMINDER_DAYS = 30` - тогда напоминания придут за 30 дней

### Вариант 2: Использовать команду /test_expiring

В боте (от админа):

## 🤖 Автоматическая работа планировщика

Планировщик работает **автоматически** вместе с ботом:

### Проверка истекших подписок:
- ⏰ Каждый день в **00:00** (полночь)
- 🔍 Находит все подписки где `end_date <= NOW()`
- 🚫 Удаляет пользователей из канала
- 📨 Отправляет уведомление "Подписка истекла"

### Уведомления о скором истечении:
- ⏰ Каждый день в **09:00** (утро)
- 🔍 Находит подписки, истекающие в течение 3 дней
- 📨 Отправляет напоминание с кнопкой "Продлить"

## 📝 Примечания:

1. **Удаление из канала работает методом ban → unban**
   - Это стандартный способ для Telegram API
   - Пользователь просто теряет доступ, но не блокируется

2. **После истечения подписка получает статус `expired`**
   - Не удаляется из БД (для истории)
   - При новой покупке создается новая запись

3. **Планировщик работает только когда бот запущен**
   - Используйте systemd или docker для постоянной работы
   - Или запускайте в screen/tmux сессии

4. **Уведомления о скором истечении отправляются ОДИН раз**
   - За REMINDER_DAYS дней (по умолчанию 3)
   - Если нужно несколько напоминаний - добавьте доп. проверки
